# tests/paper/fixtures — the frozen fixture corpus

**Freeze policy (CONVENTIONS §4).** The corpus is frozen via `MANIFEST.sha256`
(`tests/paper/test_manifest.py` enforces it). Fixtures are NEVER regenerated by
code under test. Regeneration is an explicit human decision: run
`generators/generate_corpus.py`, review every changed byte, and commit the new
manifest alongside. Each fixture has a JSON sidecar (pinned schema) next to it.

**Provenance honesty.** This environment cannot run desktop Excel. Every
fixture here is authored by stock openpyxl from this checkout, by zip surgery
on such files, or by LibreOffice conversion — the sidecar says which, exactly.
Nothing here may ever be relabeled as Excel-authored. The real-Excel bucket
(the load-bearing one: charts with styling, pivots + caches, real VBA, shared
formulas as Excel writes them, 1904 date system, Google Sheets exports) is
requested precisely in `/FIXTURE-REQUESTS.md` and lands here with sidecars
filled in by the human who authored each file.
